- S общая квартир на этаже (м2) : 
""" Из выделенных квартир отфильтровываются все, в которых заполнен параметр ADSK_Номер помещения квартиры
Считается сумма их площадей """

- S общая всех помещений на этаже (м2) :
""" Сумма всех выделенных квартир """

- S этажа ВНС (м2) : 
- S этажа ГНС1 (м2) : 
- S этажа ГНС2 (м2) : 
- S всего корпуса ГНС1 (м2) : 
- S всего корпуса ГНС2 (м2) :

1. Плагин срабатывает когда открыт вид плана
2. После нажатия выбираются все помещения на этаже

"""
После/перед п.2 выполняется создание в спецификации таблиц с внс, гнс250, гнс370, помещения.
Они нужны для контроля пользователем и не зависят от результата отработки цифровых пунктов
СДЕЛАНО
"""

3. Помещения выбраны. Производится проверка на наличие и заполнение параметров.
	* Наличие параметра ADSK_Номер квартиры
	* Получаются уникальные значения параметра Комментарий, в View попадают только те, которые проходят
	шаблон "Вариант NN" или "Вар NN"

4. Получение зон для заданного уровня.
После выбора пользователем варианта, во View отображается состояние того, существуют ли в проекте зоны ВНС, ГНС250 и ГНС370
для данного уровня и комментария
	* Параметр Имя у зоны должен подходить под шаблон "Вариант NN" или "Вар NN", чтобы он учитывался
	* Имя типоразмера зоны ВНС - "ВНС", зоны ГНС1 - "Всего по зданию", зоны ГНС2 - "ГНС_370"
	** Перечесление состояний - "Зона корректна", "Зона отсутствует", "Ошибка в параметрах"

5. После нажатия ОК, окно закрывается, а пользователь курсором указывает точку вставки нового созданного семейства таблицы
